Foreign key to ProductSubCategory.ProductSubCategoryID. - ProductModelID - Product is a member of this product model. Foreign key to ProductModel.ProductModelID. - SellStartDate - Date the product was available for sale. - SellEndDate - Date the product was no longer available for sale. - DiscontinuedDate - Date the product was discontinued. - rowguid - ROWGUIDCOL number uniquely identifying the record. Using the index

An index is an immutable ndarray implementing an ordered, sliceable set. The basic object storing axis labels for all pandas objects. Think of it as a ‚Äòrow address‚Äô for your data frame (table). It is best practice to explicitly set the index of your dataframe, as these are commonly used as primary keys which can be used to join your dataframe to other dataframes.

The dataframe can store different types (int, string, datetime), and when importing data will automatically assign a number to each row, starting at zero and counting up. You can overwrite this, which is what we are going to do.

Filtering on Multiple Conditions

Here, we will filter on multiple conditions. Before, we filtered on rows where Color was Black. We also filtered where FinishedGoodsFlag was equal to 1. Let‚Äôs see what happens when we filter on both simultaneously.

The format for multiple conditions is:

df[ (df['col1'] == value1) & (df['col2'] == value2) ]

Or, more simply:

df[ (CONDITION 1) & (CONDITION 2) ]

Which eventually may evaluate to something like:

df[ True & False ]

‚Ä¶on a row-by-row basis. If the end result is False, the row is omitted.

Don‚Äôt forget parentheses in your conditions! This is a common mistake.
